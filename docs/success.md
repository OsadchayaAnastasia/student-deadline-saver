# Definition of Success (Критерії успіху)

Перша ітерація вважатиметься успішною, якщо досягнуто наступних показників:

1. **Технічна стабільність:** Додаток запускається у браузері без помилок у консолі.
2. **Виконання сценарію:** Користувач може пройти шлях від створення до завершення завдання без зовнішніх підказок.
3. **Швидкодія:** Час додавання одного завдання займає менше ніж 15 секунд.
4. **Збереження даних:** Після оновлення сторінки список завдань не зникає (використання LocalStorage).

## План перевірки гіпотез (High Level)

**Гіпотеза:** Студентам зручніше вести простий список, ніж користуватися складними календарями.

**Як перевіримо:**
1. Надати прототип 3-м одногрупникам.
2. Попросити їх внести свої реальні дедлайни на поточний тиждень.
3. **Критерій успіху перевірки:** Якщо 2 з 3 користувачів продовжать користуватися списком наступного дня — гіпотеза підтверджена.
4. Зібрати короткий фідбек: що заважало користуватися (наприклад, незручно вводити дату з телефону).

**Управління ризиками (Risk Management):**
* *Ризик:* Не вистачить часу на реалізацію бази даних.
* *Рішення для MVP:* Використовувати локальне сховище браузера (LocalStorage), що значно спрощує розробку, але зберігає функціональність для тесту.
