Ядро цінності (MVP)

Мінімально життєздатний продукт (MVP) фокусується виключно на функції фіксації завдань. Ми відкидаємо складні функції (календар, сповіщення, інтеграція з Moodle) у першій версії.

**Ключовий функціонал:**
1. Додавання нового завдання (Назва + Дедлайн).
2. Відображення списку всіх завдань.
3. Можливість позначити завдання як «Виконане».

## Сценарій користувача (User Scenario)

**Умова початку:** Користувач відкрив головну сторінку додатка. Список завдань порожній або містить попередні записи.

**Кроки:**
1. Користувач натискає кнопку «Додати завдання».
2. Вводить назву: "Лабораторна робота №1 з ПЗ".
3. Обирає дату дедлайну (наприклад, завтра).
4. Натискає «Зберегти».
5. Система повертає користувача до головного списку.
6. Користувач бачить нове завдання у списку.
7. Користувач натискає чекбокс (галочку) біля завдання.

**Очікуваний результат:** Завдання візуально змінюється (стає перекресленим або сірим), позначаючи успішне виконання.
